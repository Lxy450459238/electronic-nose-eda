# 电子鼻 EDA 分析系统

电子鼻传感器数据的探索性数据分析（EDA）工具。支持旧版（37传感器）和新版（49传感器）硬件**自动识别**，单文件质量评估 + 多文件批量对比双模式。

> GitHub: [https://github.com/Lxy450459238/electronic-nose-eda](https://github.com/Lxy450459238/electronic-nose-eda)

---

## 系统架构概览

### single.py — 单文件 EDA（6 层流水线）

```
输入层 → 解析预处理层 → 硬件环境体检层 → 核心分析层 → 综合质量评分层 → 结果输出层
```

| 层级 | 模块 | 核心功能 |
|------|------|---------|
| 一、输入层 | `SingleFileEDA(file_path)` | 单个 txt 文件载入 |
| 二、解析预处理层 | `parse_file()` | 版本识别、元数据解析、通道拆分、首行零值裁切、阶段识别 |
| 三、硬件环境体检层 | `analyze_environment()` | 供电电压/温度/湿度/气压四维校验，异常置顶警告 |
| 四、核心分析层 | 4 个子模块顺序执行 | ①基线稳定性(四维判定ABCD) → ②进样阶段分析(P15→P17回退,滑动窗口防伪稳态) → ③时间序列相关性(皮尔逊矩阵) → ④清洗阶段分析(反向越界二级分类) |
| 五、综合质量评分层 | `calculate_quality_score()` | 基础50分，分项累加，clamp[0,100]，优秀/良好/较差三级 |
| 六、结果输出层 | `generate_report()` + `visualize_all()` | 文本报告(.txt) + 5张可视化图表 |

**四维基线判定 (ABCD)**：A)大均值CV B)小均值STD兜底 C)全局STD红线 D)PTP极差漂移

### multi.py — 多文件 EDA（8 层流水线，支持双模式自适应）

```
输入层 → 批量数据加载层 → 环境体检层 → 多样性判定层 → 核心统计层 → 异常检测层 → 相关性分析层 → 结果输出层
```

| 层级 | 模块 | 核心功能 |
|------|------|---------|
| 一、输入层 | 文件夹扫描 | `glob("*.txt")` 批量载入 |
| 二、数据加载层 | `load_and_extract_data()` | 版本自动识别(首个文件)、元数据解析(正则日期+分隔符)、首行零值裁切、通道拆分(传感器/环境分离)、**P15优先→P17动态回退**、稳态值批量提取、容错跳过 |
| 三、环境体检层 | `analyze_environment()` | 基于首个样本全周期数据，四维校验(电压/温度/湿度/气压)，异常汇总→报告置顶 |
| 四、多样性判定层 | `check_diversity()` | 气体种类+浓度水平 → 二分类(低多样性/高多样性)，**模式全局自适应** |
| 五、核心统计层 | `analyze_statistics()` | **分支A(低多样性)**：双维度稳定性判定(CV为主, 绝对STD兜底)，稳定/一般/不稳定三级；**分支B(高多样性)**：SNR+F值双指标，95分位数自适应归一化，强烈推荐/推荐/可保留/建议剔除四级 |
| 六、异常检测层 | `detect_outliers()` | 1.5×IQR箱线图法，≥5传感器同时异常→严重异常标记 |
| 七、相关性分析层 | `calculate_correlation()` | **分支A**：随机抽样→进样前半段动态数据→多样本矩阵平均→\|r\|>0.8筛选；**分支B**：稳态皮尔逊矩阵→\|r\|>0.8筛选 |
| 八、结果输出层 | `generate_report()` + `visualize_all()` | 文本报告(硬件体检置顶) + 自适应可视化(通用图 + 模式专属图) |

**架构图**：[单文件架构](plot_scripts/eda_architecture.drawio) | [多文件架构](plot_scripts/multi_eda_architecture.drawio)

## 一键复现指南

### 第 1 步：获取代码

```bash
git clone https://github.com/Lxy450459238/electronic-nose-eda.git
cd electronic-nose-eda/EDA(6.8)
```

### 第 2 步：安装环境

Python 3.8+，安装依赖：

```bash
pip install -r requirements.txt
```

依赖：`numpy` `pandas` `matplotlib` `seaborn` `scipy`

### 第 3 步：准备数据

将 `.txt` 数据文件放到任意目录。数据要求：

- 空格分隔的文本文件
- 第 5 列为阶段标签（P13 / P14 / P15 / P16 / P17 / P18）
- 旧版电子鼻（37 传感器）：至少 51 列
- 新版电子鼻（49 传感器）：至少 83 列
- 文件名建议遵循 `日期-编号-气体-浓度-重复次数.txt` 格式

### 第 4 步：运行分析

**单文件分析：**

```bash
python single.py <数据文件路径>
```

输出 5 张图 + 1 份文本报告 → `EDATestResult/`

**多文件批量分析：**

```bash
python multi.py <数据文件夹路径>
```

输出 6 张图（模式自适应） + 1 份文本报告 → `EDATestResult/`

### 第 5 步：查看结果

**单文件输出** (`EDATestResult/`)：

### 常见问题

| 问题 | 解决方法 |
|------|---------|
| 报错 `找不到文件` | 用引号包裹含括号/空格的路径 |
| 报错 `文件夹中没有找到txt文件` | 确认文件夹内有 `.txt` 文件 |
| 报错 `数据文件列数不足` | 旧版需 ≥51 列，新版需 ≥83 列 |
| 中文图表乱码 | 确保 `C:\Windows\Fonts\simhei.ttf` 存在 |
| 多文件报 `zero-size array` | 旧版需 P15 数据；最新版已支持 P15→P17 自动回退 |

## 关键设计要点

### P15/P17 动态进样阶段匹配

`single.py` 和 `multi.py` 均已同步支持：
- **P15 优先**：存在 P15 数据时使用 P15 作为进样阶段
- **P17 回退**：无 P15 时自动回退使用 P17
- **P15+P17 共存**：额外保存 P17 稳态值，预留一致性校验

### 双维度稳定性判定（多文件低多样性场景）

避免小均值下 CV 失真：

| 场景 | 条件 | 判定规则 |
|------|------|---------|
| 大均值 | \|μ\| > 20 | CV < 0.1 稳定 / CV > 0.3 不稳定 / 0.1~0.3 一般 |
| 小均值 | \|μ\| ≤ 20 | σ ≤ 20 稳定 / σ > 20 不稳定（绝对标准差兜底） |

### 自适应归一化（多文件高多样性场景）

SNR 和 F 值使用 **95 分位数** 作为归一化因子，削弱极端值对综合评分的影响，使区分度排名更稳健。
