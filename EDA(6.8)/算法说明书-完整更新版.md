## 摘要

本文档是面向实验室自主研发通用电子鼻系统的**自动化探索性数据分析（EDA）算法规格说明书**。系统设计为双层架构——单文件EDA（S-EDA）对单次实验数据进行深度质量评估，输出0–100量化评分；多文件EDA（M-EDA）对批量数据进行统计对比，评估传感器阵列的稳定性与气体区分能力。

相较于旧版说明书（基于 v3 代码），本文档以 EDA(6.8) 最终整理版代码为唯一真相源，系统性更新了以下核心内容：（1）**四维基线稳定性融合判别准则**，以极差漂移、全局STD红线、大均值CV和小均值STD兜底四重规则替代单一CV判定；（2）**硬件环境体检模块**，将供电电压、温度、湿度、气压等物理通道独立于气体化学通道进行绝对区间校验；（3）**自适应进样/清洗阶段路由**，动态探测P15/P17进样阶段并自动匹配P16/P18清洗阶段；（4）**95分位数自适应归一化区分度评分**，替代旧版固定因子方案；（5）**多段式传感器数据提取架构**，支持新版49传感器五段碎片化列区间精准剥离；（6）**硬件版本自动识别**，通过列数判定自动加载37或49传感器配置；（7）**智能日期解析**，以正则匹配替代简单字符串切割，兼容非标准文件命名格式。文档包含21处公式位置和13处图片预留位，可作为重写正式 docx 说明书的完整蓝本。

# 第一部分：系统概述

## 1. 引言

### 1.1 背景与意义

在现代嗅觉计算与气体传感器阵列技术（即"电子鼻"）的研究与应用中，数据质量的预处理与探索性数据分析（Exploratory Data Analysis, EDA）构成了整个信号处理链路的基石。本实验室研制的通用电子鼻系统由多种金属氧化物半导体（MOS）传感器、电化学传感器、PID传感器及红外光学传感器组成的阵列构成，这些传感器在对目标气体产生响应的同时，也深受环境温湿度、基线漂移（Drift）、传感器老化以及交叉敏感度（Cross-sensitivity）的影响。

本说明书详细阐述针对通用电子鼻系统开发的自动化EDA算法逻辑。该系统为**双层架构**：

- **单文件EDA（Single-File EDA，S-EDA）**：对单次实验数据进行深度质量评估，输出量化质量评分。
- **多文件EDA（Multi-File EDA，M-EDA）**：对批量实验数据进行统计对比，评估传感器稳定性和区分度。

系统支持**新旧两代硬件**的自动识别：旧版搭载37个气体传感器，新版搭载49个纯气体传感器。

### 1.2 适用范围

本系统适用于采用标准时间序列采集模式的通用型电子鼻，涉及多阶段（基线→进样→清洗）循环工作流程。算法逻辑基于传感器响应的统计学特性和物理学规律，具有广泛的通用性。

## 2. 数据架构与采集规范

### 2.1 实验阶段定义（P13–P18）

电子鼻一次完整采样过程由若干功能各异的物理阶段组成。算法依据第5列（索引4）的阶段标签将原始时间序列划分为以下关键区间：

| 阶段代码 | 阶段名称 | 物理含义与算法用途 | 典型流速配置 |
|----------|----------|-------------------|-------------|
| **P13** | 基线阶段 (Primary Baseline) | 传感器在洁净空气中的初始响应。是判断系统是否就绪的关键指标。 | 与进样一致 |
| **P14** | 空气前进样阶段 (Pre-Sampling) | 紧邻样品注入前的二次基线确认。当P13不稳定时自动切换至此阶段。 | 与进样一致 |
| **P15** | 加工前进样阶段 (Sampling) | 传感器暴露于目标分析物的核心响应阶段。提取稳态响应值、时间序列相关性。 | 与进样一致 |
| **P16** | 清洗阶段 (Cleaning) | 通入洁净空气进行脱附。计算残留率和清洗效率，决定记忆效应风险。 | 高流速冲洗 |
| **P17** | 加工后进样阶段 (Post-Sampling) | 可选的二次进样验证阶段，用于对比实验或长周期观测。 | 与进样一致 |
| **P18** | 二次清洗阶段 (Cleaning) | 最终的系统吹扫。 | 高流速冲洗 |

> **注：** 一次完整采样只需包含必须的基线-进样-清洗流程，不一定包含所有六个阶段。算法会自动探测存在的阶段并动态适配分析路径。

### 2.2 标准化文件命名协议 ✏️ 更新

文件命名格式：`日期-编号-气体名称-浓度-重复次数.txt`

**标准格式示例：** `20230320-302-bingtong-30ppm-1.txt`

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 日期 | 20230320 | 8位日期（YYYYMMDD），用于追踪传感器老化趋势 |
| 编号 | 302 | 实验室内部气体ID |
| 气体名称 | bingtong | 丙酮。多文件EDA的分类标签 |
| 浓度 | 30ppm | 定量参数，SNR计算中"信号"的主要来源 |
| 重复次数 | 1 | 同条件重复实验编号，用于组内方差计算 |

**🆕 智能日期提取：** 系统使用正则表达式 `r'20\d{6}'` 从文件名任意位置提取8位连续日期数字，兼容非标准命名格式。

**非标准格式示例：** `yangji1-034-huangqin(26)-2408-830-20211013_20260306094501.txt`
→ 系统自动提取日期为 `20211013`，而非错误地取 `yangji1`。

## 3. 🆕 数据预处理

### 3.1 首行零值跳过

电子鼻上电初期，传感器加热丝可能尚未达到热平衡，导致前几行气体传感器读数为0（硬件通信未就绪）。

**处理逻辑：**
- 检查前 `DATA_SKIP_ROWS_MAX`（默认3）行
- 仅检查前 `DATA_SKIP_CHECK_COLS`（默认37）个核心气体传感器列
- 只要某行核心传感器存在0值，即判定该行需跳过
- 遇到不包含0值的行立即停止检查 →

**关键：同步裁切** — 跳过行时，同时裁切传感器数据、时间戳数组和原始索引，确保底层数据结构完全对齐。single.py 和 multi.py 均实现了此逻辑。

## 4. ✏️ 动态基线稳定性分析（四维融合判定）

### 4.1 变异系数（CV）基础公式

对于任意传感器，在选定的基线时间窗口内采集 *n* 个数据点：

**[公式]**均值：$$\mu = \frac{1}{n}\sum_{i=1}^{n} x_i$$

**[公式]**标准差：$$\sigma = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(x_i - \mu)^2}$$

**[公式]**变异系数：$$CV = \frac{\sigma}{|\mu| + \epsilon}, \quad \epsilon = 10^{-6}$$

### 4.2 ✏️🆕 四维融合判定规则

**当前代码实现四维判定准则，任一维度触碰红线即判定为不稳定：**

| 维度 | 规则名称 | 触发条件 | 适用场景 |
|------|---------|---------|---------|
| **A** | 大均值CV规则 | \|μ\| ≥ 50 且 CV ≥ 0.15 | 基准读数较高的传感器 |
| **B** | 小均值STD兜底 | \|μ\| < 50 且 σ ≥ 20.0 | 差分输出/归零传感器（防CV数学爆炸误杀） |
| **C** | 全局大基数STD | σ ≥ 30.0（任何传感器） | CO2等极大基数传感器补充规则 |
| **D** | 极差漂移规则 | max−min ≥ 120.0 | 抓捕"过山车式"剧烈起伏和单向严重漂移 |

> **严重程度优先级：** 极差 > 全局STD > CV > 小均值STD
> 报告输出时按优先级显示具体"死因参数"。

**允许的不稳定传感器上限：** 动态比例 = 传感器总数 × `BASELINE_UNSTABLE_RATIO`（默认0.15）
- 49传感器版：允许 ≤7个不稳定
- 37传感器版：允许 ≤5个不稳定

### 4.3 P13/P14 自适应切换逻辑

```
1. 优先评估 P13，计算四维判定下"不稳定"传感器数 N_unstable
2. 若 N_unstable ≤ max_unstable_allowed → 锁定 baseline_stage = 'P13'
3. 若 N_unstable > max_unstable_allowed → 检查 P14 是否存在
   a) P14 存在 → 切换到 baseline_stage = 'P14'（重新判定）
   b) P14 不存在 → 继续使用 P13，发出警告，质量评分 -15
```

### 4.4 基线异常值检查（IQR离群点分析）✏️ 更新

针对选定基线阶段的每个传感器，采用 **3倍IQR** 边界进行毛刺检测：

**[公式]**下四分位数 *Q1*：25%分位点
**[公式]**上四分位数 *Q3*：75%分位点
**[公式]**四分位距：*IQR = Q3 − Q1*

**[公式]**异常边界：
- 下限 = *Q1 − 3.0 × IQR*
- 上限 = *Q3 + 3.0 × IQR*

> **为何3倍而非1.5倍：** 3×IQR 对应正态分布下的极小概率事件，确保仅抓取非化学性质的电子毛刺（Spikes），避免误杀正常热噪声。

**🆕 精细化单通道占比统计（区别于旧版的全局计数）：**

对每个传感器单独计算异常点占比：
- 若某传感器异常点占比 > `BASELINE_OUTLIER_RATIO_THRESHOLD`（默认5%）
- → 该传感器被标记为"持续性毛刺干扰"，在报告中单独点名
- → 触发质量评分 **−10**

## 5. ✏️ 响应阶段的稳态搜索与判定

### 5.1 滑动窗口稳态检测算法

**[公式]**窗口大小 *W* = min(10, *n*/4)，其中 *n* 为该阶段数据点总数。

**判别标准：** 窗口内标准差 < |窗口均值| × 5%

**前瞻验证：** 找到候选稳态窗口后，继续验证后续20个数据点的相邻差分是否均满足稳定性要求。只有通过二次校验，窗口均值才被确认为稳态响应值。

**回退机制：** 若整个序列无法满足条件，取后半段数据（后50%）的均值作为估算值。若超过10个传感器触发回退，系统发出警告。

### 5.2 响应比率与有效传感器判定

**[公式]**响应比率：$$R = \frac{|\mu_{sample} - \mu_{baseline}|}{|\mu_{baseline}| + \epsilon}$$

- *R* > 0.1 → **有效响应传感器**（响应幅度超过基线10%）
- *R* ≤ 0.1 → 无效响应（视为噪声或漂移）

> **阈值依据：** MOS传感器的典型响应幅度在10%–50%之间。若基线固有噪声已达10%，真实气体响应信号将被完全淹没（SNR→1:1），该传感器物理上丧失检测能力。

**最少响应传感器数：** 要求至少 `RESPONSE_MIN_COUNT`（默认10）个传感器产生有效响应。若不足 → **−15分**。

**[插入图片位置：响应增量柱状图，绿色=正向响应，红色=负向响应]**

### 5.3 ✏️🆕 自适应进样阶段路由

**旧版逻辑（已废弃）：** 硬编码固定寻找P15作为进样阶段。

**新版逻辑：** 动态探测：

```
优先探测 P15 → 若存在则作为 sample_stage
否则探测 P17 → 若存在则作为 sample_stage
否则报告"未找到进样阶段"并跳过进样分析
```

**双气室一致性验证（P15+P17同时存在时）：**
- 计算P15和P17稳态响应值的皮尔逊相关系数 *r*
- *r* > 0.95 → 两次进样高度一致，**+10分**
- *r* < 0.7 → 两次进样差异显著，**−10分**

## 6. ✏️ 清洗效率与残留率评估

### 6.1 残留率计算公式

**[公式]**残留率：$$RecoveryRate = \frac{|\mu_{clean} - \mu_{baseline}|}{|\mu_{sample} - \mu_{baseline}|}$$

其中：
- *μ_baseline*：基线阶段稳态均值
- *μ_sample*：进样阶段稳态均值
- *μ_clean*：清洗阶段末端稳态均值

**归一化特性：** 分母为总响应幅度，分子为残留幅度。结果在0–1之间，对不同浓度和灵敏度的传感器提供可比数值。

**越界处理：** 若符号反转（*residual × response_amplitude < 0*），说明已越过基线 → 残留率设为0，标记为"过度清洗或漂移"。

### 6.2 清洗质量分级标准

| 等级 | 残留率范围 | 含义 |
|------|-----------|------|
| 优秀 (Excellent) | < 5% | 完全脱附 |
| 良好 (Good) | 5%–15% | 基本恢复 |
| 较差 (Poor) | ≥ 25% | 记忆效应风险 |

**仅评估有效响应传感器：** 配置项 `CLEANING_EVAL_RESPONSIVE_ONLY = True`，未响应的传感器不参与残留率计算，避免分母为0。

**全局判定：**
- 优+良传感器比例 ≥ 80%（`CLEANING_GOOD_RATIO`）→ 清洗充分，**+10分**
- 平均残留率 ≤ 15% → 清洗可接受，**+5分**
- 不满足以上 → 清洗不充分，**−10分**

## 7. 时间序列相关性与冗余分析

### 7.1 相关系数矩阵

**[公式]**皮尔逊相关系数：$$r_{ij} = \frac{\sum(x_i - \bar{x}_i)(x_j - \bar{x}_j)}{\sqrt{\sum(x_i - \bar{x}_i)^2 \sum(x_j - \bar{x}_j)^2}}$$

**数据选取：** 截取进样阶段前50%（`CORRELATION_DYNAMIC_RATIO = 0.5`）的"动态爬坡期"数据计算，捕捉传感器响应动力学特征。

**高相关阈值：** |*r*| > 0.8 的传感器对被视为高度冗余。

## 8. 综合质量评分模型 (Quality Score Model)

**基础分：50分**

| 评估维度 | 触发条件 | 分值调整 |
|----------|---------|---------|
| 基线稳定 | 选定基线的不稳定传感器数 ≤ 上限 | **+20** |
| 基线不稳定 | 不稳定传感器超过上限 | **−15** |
| 基线毛刺 | 存在持续性毛刺干扰通道 | **−10** |
| 响应丰富 | 有效响应传感器 ≥ 10个 | **+25** |
| 响应匮乏 | 有效响应传感器 < 10个 | **−15** |
| 进样一致 | P15与P17相关系数 > 0.95 | **+10** |
| 进样分歧 | P15与P17相关系数 < 0.7 | **−10** |
| 清洗充分 | 优+良传感器 ≥ 80% | **+10** |
| 清洗可接受 | 平均残留率 ≤ 15% | **+5** |
| 清洗不足 | 不满足以上 | **−10** |

**评分应用：**
- ≥ 80分 → 优秀，建议直接入库
- 60–80分 → 良好，需注意警告信息
- < 60分 → 较差，建议人工复查或剔除

## 10. 样本多样性自动检测

扫描所有输入文件的元数据，根据样本构成自动分流算法路径：

| 场景 | 判定条件 | 分析目标 |
|------|---------|---------|
| 低多样性 | 气体种类数 = 1 且 浓度水平数 = 1 | **稳定性 (Stability)** |
| 高多样性 | 气体种类或浓度不唯一 | **区分度 (Discrimination)** |

## 11. 稳定性评估（低多样性模式）

**数据聚合：** 提取N个文件中同一传感器的稳态响应值。

**[公式]**稳定性CV：$$CV_{stability} = \frac{\sigma_{sensor}}{\mu_{sensor}}$$

**分类标准：**

| 等级 | CV范围 | 含义 |
|------|--------|------|
| 高稳定 | CV < 0.1 | 重复实验中输出一致，定量分析首选 |
| 不稳定 | CV > 0.3 | 响应不可复现，建议建模时剔除 |
| 灰色地带 | 0.1 ≤ CV ≤ 0.3 | 标记为"一般" |

**[插入图片位置：稳定性统计表（绿色/黄色/红色标注）]**

## 12. ✏️ 区分度评估（高多样性模式）

### 12.1 信噪比（SNR）计算

将数据按标签（气体+浓度）分组，每组为相同条件下的重复实验。

**[公式]**信号项：$$Max\_diff = \max_k(\mu_k) - \min_k(\mu_k)$$

其中 *μ_k* 为第 *k* 组的均值。

**[公式]**噪声项：$$Avg\_std = \frac{1}{K}\sum_{k=1}^{K} \sigma_k$$

其中 *σ_k* 为第 *k* 组的组内标准差。

**[公式]**SNR：$$SNR = \frac{Max\_diff}{Avg\_std + \epsilon}$$

### 12.2 ANOVA F-value

对每个传感器执行单因素方差分析（One-way ANOVA）：

**[公式]**$$F = \frac{MS_{between}}{MS_{within}} = \frac{组间方差}{组内方差}$$

p值越小 → 拒绝"各组均值相同"原假设的信心越强。

### 12.3 ✏️🆕 自适应归一化混合评分

**旧版（固定因子）：** 使用硬编码 SNR_norm = min(SNR/10.0, 1.0), F_norm = min(F/50.0, 1.0)

**🆕 新版（自适应95分位数）：**

**[公式]**$$SNR_{norm} = \min\left(\frac{SNR}{P_{95}(SNR)}, 1.0\right)$$

**[公式]**$$F_{norm} = \min\left(\frac{F}{P_{95}(F)}, 1.0\right)$$

其中 *P_95* 为所有传感器该指标的95分位数。自适应方案消除了硬件版本差异对固定因子的影响。

**[公式]**综合区分度评分：$$Score = 0.6 \times SNR_{norm} + 0.4 \times F_{norm}$$

**保留建议判据：**

| 等级 | 条件 | 
|------|------|
| 强烈推荐保留 | Score > 0.7 且 p < 0.001 |
| 推荐保留 | Score > 0.5 且 p < 0.01 |
| 可以保留 | Score > 0.3 |
| 建议剔除 | 其他情况 |

**[插入图片位置：区分度排名水平柱状图（Top 20），绿/橙/红三色标注]**

## 13. 异常样本检测

对于每个传感器，收集所有N个样本的稳态响应值：

**[公式]**IQR = *Q3 − Q1*
**[公式]**异常边界：下限 = *Q1 − 1.5×IQR*，上限 = *Q3 + 1.5×IQR*

**判据：** 若一个样本在超过 `OUTLIER_MIN_ABNORMAL_SENSORS`（默认5）个传感器上均被判定为异常，则该样本标记为"严重异常样本"。

**[插入图片位置：箱线图，标注异常样本]**

## 14. 传感器相关性分析

**低多样性：** 随机抽取10个代表性样本，计算P15动态阶段的时间序列相关系数矩阵，取平均值。

**高多样性：** 基于所有样本的稳态响应值计算相关系数矩阵。

两种模式下均以 |*r*| > 0.8 为高相关阈值，识别冗余传感器对。
