## 二、config.py —— UniversalConfig（纯静态配置类）

所有方法均为 `@classmethod`，无需实例化，直接通过 `Config.method()` 调用。

### 2.1 `detect_version()`

**功能说明：**
根据数据文件的总列数自动判定电子鼻硬件版本（旧版37传感器 / 新版49传感器）。判定阈值由 `OLD_VERSION['min_columns']`（51列）和 `NEW_VERSION['min_columns']`（83列）定义。

**流程图：**
```
读入数据 → 获取总列数 N
              │
     ┌───────┼───────┐
     ▼       ▼       ▼
  N ≥ 83  51≤N<83  N < 51
     │       │       │
     ▼       ▼       ▼
  新版49   旧版37  抛出 ValueError
 传感器   传感器   "数据文件列数不足"
```

**入口参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `data_columns` | `int` | 数据文件的总列数，由 `pd.read_csv(...).shape[1]` 获得 |

**出口参数：**

| 返回 | 类型 | 说明 |
|------|------|------|
| `version_config` | `dict` | 命中版本的完整配置字典，包含 `name`, `sensor_count`, `sensor_config`, `env_config`, `env_name_map`, `sensor_name_map`, `min_columns` |

**调用方式：**
```python
from config import UniversalConfig as Config
data_df = pd.read_csv(file_path, sep=' ', header=None)
version_config = Config.detect_version(data_df.shape[1])
```

### 2.2 `extract_sensor_data()`

**功能说明：**
根据版本配置中的 `sensor_config` 字典，从原始DataFrame中按列区间切片提取纯气体传感器数据。新版49传感器采用五段碎片化提取（Part1–Part5），旧版37传感器为单段连续提取。提取完成后校验传感器数量是否与配置一致。

**流程图：**
```
读入 data_df + version_config
         │
         ▼
遍历 sensor_config 字典 (key: 段名, value: (start_col, end_col))
         │
         ▼ 对每段执行
   df.iloc[:, start:end] → ndarray
         │
         ▼
np.concatenate() 合并所有段 → (n_points, n_sensors)
         │
         ▼
校验 shape[1] == version_config['sensor_count']
   不匹配 → 抛出 ValueError
```

**入口参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `data_df` | `pd.DataFrame` | 空格分隔读取的原始数据 |
| `version_config` | `dict` | `detect_version()` 返回的版本配置 |

**出口参数：**

| 返回 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `sensor_data` | `np.ndarray` | `(n_samples, n_sensors)` | 纯净气体传感器数值矩阵 |

**调用方式：**
```python
sensor_data = Config.extract_sensor_data(data_df, version_config)
```

### 2.3 `extract_env_data()`

**功能说明：**
从原始DataFrame中独立提取环境/物理监控通道数据（温度、湿度、气压、供电电压），与气体化学通道分离管理。旧版提取9个通道，新版提取11个通道（含扩展温湿度）。

**入口参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `data_df` | `pd.DataFrame` | 原始数据 |
| `version_config` | `dict` | 版本配置，需包含 `env_config` 键 |

**出口参数：**

| 返回 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `env_data` | `np.ndarray` 或 `None` | `(n_samples, n_env_channels)` | 环境通道矩阵；若配置无 `env_config` 则返回 `None` |

**调用方式：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `file_path` | `str` | — | .txt 数据文件的绝对或相对路径 |

### 3.2 `parse_file(self)`

**功能说明：**
数据加载与预处理的总入口。依次完成：空格分隔读取txt → 列数检测 → 硬件版本识别 → 传感器数据提取 → 环境数据提取 → 阶段标签切分（P13–P18）→ 首行零值裁切 → 元数据解析。执行后 `self.stages`、`self.version_config`、`self.num_sensors` 等属性被填充。

**流程图：**
```
pd.read_csv(file, sep=' ', header=None)
         │
         ▼
data.shape[1] → Config.detect_version() → version_config
         │
         ▼
Config.extract_sensor_data() → sensor_data (n×m)
Config.extract_env_data()    → env_data (n×k)
         │
         ▼
遍历 P13, P14, P15, P16, P17, P18:
   stage_column == 'Pxx' → 布尔索引 → 切片 sensor_data
         │
         ▼
首行零值检查（前3行核心传感器列含0则裁切）
         │
         ▼
_parse_metadata() → 从文件名解析气体/浓度/日期
         │
         ▼
self.stages = { 'P13': {data, time, indices}, 'P14': {...}, ... }
```

**入口参数：** 无（从 `self.file_path` 读取）。

**出口参数：** 无返回值，填充以下属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| `self.data` | `pd.DataFrame` | 原始完整数据 |
| `self.version_config` | `dict` | 版本配置 |
| `self.num_sensors` | `int` | 传感器数量 |
| `self.stages` | `dict` | 阶段→数据映射 |
| `self.env_data` | `np.ndarray` 或 `None` | 环境通道数据 |
| `self.metadata` | `dict` | 元数据 |

**调用方式：**
```python
eda.parse_file()
# 该方法由 run_full_analysis() 内部自动调用，通常无需手动调用
```

### 3.3 `_parse_metadata(self)`

**功能说明：**
从文件名中智能解析实验元数据。采用正则 `r'20\d{6}'` 提取8位日期（YYYYMMDD），按 `-` 分割文件名后按位置和关键词匹配气体类型与浓度。兼容标准命名（`日期-编号-气体-浓度-重复次数`）和非标准命名（含前缀/后缀）。

**流程图：**
```
文件名 "20230320-302-bingtong-30ppm-1.txt"
         │
         ├─ re.search(r'20\d{6}', filename) → "20230320"
         │
         ├─ split('-') → ['20230320','302','bingtong','30ppm','1.txt']
         │        │
         │        ├─ 遍历parts，匹配气体名（bingtong/ethanol/...）
         │        └─ 匹配浓度（含ppm/ppb/%的字段）
         │
         └─ → self.metadata = {date, gas_type, concentration, ...}
```

**入口参数：** 无（从 `self.filename` 读取）。

**出口参数：** 无返回值，填充 `self.metadata` 字典。

**调用方式：**
```python
# 由 parse_file() 内部自动调用
```

### 3.4 `analyze_baseline(self)`

**功能说明：**
执行四维融合基线稳定性判定。优先评估P13阶段，对每个传感器并行计算四个维度的红线指标：大均值CV（|μ|≥50 且 CV≥0.15）、小均值STD兜底（|μ|<50 且 σ≥20）、全局STD红线（σ≥30）、极差漂移（max−min≥120）。P13不稳定传感器数超阈值（总传感器数×15%）时自动切换P14，无P14则维持P13并扣15分。

**流程图：**
```
优先评估 P13
    │
    ├─ 计算 P13_mean, P13_std, P13_cv, P13_ptp (向量化，shape = num_sensors)
    │
    ├─ 四维判定 (向量化并行)：
    │     cv_mask      = (|μ| ≥ 50) & (CV ≥ 0.15)
    │     small_std    = (|μ| < 50) & (σ ≥ 20)
    │     large_std    = σ ≥ 30
    │     ptp_mask     = (max-min) ≥ 120
    │     unstable     = cv_mask | small_std | large_std | ptp_mask
    │
    ├─ 统计 unstable_count = sum(unstable_mask)
    │
    ├─ unstable_count ≤ max_allowed (15% × N) ?
    │       ├─ YES → baseline_stage = 'P13', +20分
    │       └─ NO  → P14 存在?
    │                  ├─ YES → baseline_stage = 'P14', 重新判定
    │                  └─ NO  → baseline_stage = 'P13', -15分, 警告
    │
    ├─ IQR离群点检测 (3×IQR, 单通道占比阈值5%) → 毛刺传感器
    │
    └─ 结果写入 self.results['baseline']
```

**入口参数：** 无（从 `self.stages` 读取）。

**出口参数：** 无返回值（链式 `return self`），写入 `self.results['baseline']`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `stage` | `str` | 最终选定的基线阶段名 |
| `mean` | `np.ndarray` | 各传感器基线均值 `(num_sensors,)` |
| `std` | `np.ndarray` | 各传感器基线标准差 |
| `cv` | `np.ndarray` | 各传感器变异系数 |
| `ptp` | `np.ndarray` | 各传感器极差 |
| `unstable_mask` | `np.ndarray(bool)` | 不稳定传感器布尔掩码 |
| `outlier_info` | `dict` | 离群点详细信息 |

副作用：修改 `self.quality_score`（+20 / −15 / −10）、`self.warnings`。

**调用方式：**
```python
eda.analyze_baseline()
# 必须在 parse_file() 之后调用
```

### 3.5 `extract_steady_state(self, stage_data, stage_name)`

**功能说明：**
通用滑动窗口稳态检测算法。从左向右扫描传感器时间序列，以窗口大小 `min(10, n/4)` 滑动搜索满足 `σ < |μ|×5%` 的候选窗口，命中后前瞻验证后续20个点的相邻差分是否也满足5%条件。双条件均满足则锁定该窗口为稳态段，取其均值作为稳态响应值。遍历完毕无满足条件时回退取后50%数据均值。对应原理示意图：`plot_scripts/steady_state_schematic.png`。

**流程图：**
```
对每个传感器 sensor_series (长度 n):
    │
    ├─ n < 30 → 取全段均值，跳过
    │
    ├─ window_size = min(10, n//4)
    │
    ├─ for i in 0 .. (n - window_size - 20):     ← 从左向右滑动
    │      window = series[i : i+window_size]
    │      │
    │      ├─ σ(window) < |μ(window)| × 5% ?
    │      │     ├─ YES → next_20 = series[i+window_size : i+window_size+20]
    │      │     │        所有|相邻差分| < |μ(next_20)| × 5% ?
    │      │     │          ├─ YES → 锁定稳态段 = window + next_20 → break
    │      │     │          └─ NO  → 继续滑动
    │      │     └─ NO  → 继续滑动
    │      │
    │      └─ 遍历完毕未锁定 → 回退: steady_segment = series[n//2 :]
    │
    └─ 返回 mean(steady_segment)
```

**入口参数：**

| 参数 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `stage_data` | `np.ndarray` | `(n_points, num_sensors)` | 某阶段全部传感器数据 |
| `stage_name` | `str` | — | 阶段名（仅用于日志打印） |

**出口参数：**

| 返回 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `steady_values` | `np.ndarray` | `(num_sensors,)` | 各传感器在该阶段的稳态响应均值 |

副作用：若超过10个传感器未检测到稳态段，追加警告到 `self.warnings`。

**调用方式：**
```python
sample_steady = eda.extract_steady_state(P15_data, 'P15')
# 通常由 analyze_sampling() 内部调用
```

### 3.6 `analyze_sampling(self)`

**功能说明：**
进样阶段分析的总入口。优先探测P15作为核心进样阶段，P15不存在时回退到P17。调用 `extract_steady_state()` 获取稳态响应值后，计算各传感器的响应比率 `R = |μ_sample − μ_baseline| / (|μ_baseline| + ε)`，R>0.1判定为有效响应传感器。若P15和P17同时存在，额外计算双气室一致性相关系数。最少要求10个传感器产生有效响应。

**流程图：**
```
sample_stage = 'P15' if P15存在 else 'P17' if P17存在 else None
    │
    ├─ sample_stage 为 None → 警告，跳过
    │
    ├─ extract_steady_state(sample_data, sample_stage) → steady_values
    │
    ├─ 响应比率计算:
    │     response_delta = steady - baseline
    │     response_ratio = |delta| / (|baseline| + ε)
    │     responsive = ratio > 0.1
    │
    ├─ responsive_count:
    │     ≥ 10 → +25分
    │     < 10 → -15分
    │
    └─ P15 & P17 同时存在?
          ├─ extract_steady_state(P17) → P17_steady
          ├─ corrcoef(P15_steady, P17_steady)
          ├─ r > 0.95 → +10分
          └─ r < 0.7  → -10分
```

**入口参数：** 无（从 `self.stages` 和 `self.results['baseline']` 读取）。

**出口参数：** 无返回值（链式 `return self`），写入 `self.results`：

| 字段 | 说明 |
|------|------|
| `sample_stage` | 选定的进样阶段名 |
| `sample_steady` | 各传感器稳态响应值 `(num_sensors,)` |
| `sample_response['delta']` | 响应增量 |
| `sample_response['ratio']` | 响应比率 |
| `sample_response['responsive_count']` | 有效响应传感器数 |

副作用：设置 `self.responsive_sensors`、`self.non_responsive_sensors`，修改 `self.quality_score`。

**调用方式：**
```python
eda.analyze_sampling()
# 必须在 analyze_baseline() 之后调用
```

### 3.7 `calculate_temporal_correlation(self)`

**功能说明：**
计算进样阶段前50%（"动态爬坡期"）各传感器之间的皮尔逊相关系数矩阵。截取前半段数据的目的是捕捉传感器在响应上升阶段的动力学协同特征，而非稳态平台的共性。|r| > 0.8 的传感器对被记录为高度冗余。

**入口参数：** 无（从 `self.stages[sample_stage]` 读取）。

**出口参数：** 写入 `self.results['temporal_correlation']`：

| 字段 | 说明 |
|------|------|
| `matrix` | 相关系数矩阵 `(num_sensors, num_sensors)` |
| `high_corr_pairs` | `[(sensor_i, sensor_j, r_value), ...]` 高相关对列表 |

**调用方式：**
```python
eda.calculate_temporal_correlation()
```

### 3.8 `extract_cleaning_steady_state(self, stage_data, baseline)`

**功能说明：**
清洗阶段专用稳态搜索算法。与进样阶段的 `extract_steady_state()` 不同，此方法**从序列末端向前搜索**（清洗阶段末端更接近稳态），窗口大小 `min(10, n/5)`，稳态判定条件为窗口标准差 < 基线绝对值的5%。未找到稳态时回退取后40%数据均值。

**流程图：**
```
对每个传感器 sensor_series (长度 n):
    │
    ├─ n < 20 → 取后半段均值，跳过
    │
    ├─ for i in (n-window) .. n//2  (从末端向前):
    │      window = series[i : i+window_size]
    │      σ(window) < |baseline| × 5% ?
    │        ├─ YES → 锁定, break
    │        └─ NO  → 继续向前
    │
    └─ 未锁定 → 回退取 series[n*0.6 : ] 均值
```

**入口参数：**

| 参数 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `stage_data` | `np.ndarray` | `(n_points, num_sensors)` | 清洗阶段数据 |
| `baseline` | `np.ndarray` | `(num_sensors,)` | 基线均值（来自 `self.results['baseline']['mean']`） |

**出口参数：**

| 返回 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `steady_values` | `np.ndarray` | `(num_sensors,)` | 各传感器清洗后稳态值 |

**调用方式：**
```python
clean_steady = eda.extract_cleaning_steady_state(P16_data, baseline_mean)
```

### 3.9 `analyze_cleaning(self)`

**功能说明：**
清洗效率评估的总入口。根据进样阶段动态匹配对应的清洗阶段（P15→优先P16降级P18，P17→P18）。计算残留率 `RR = |μ_clean − μ_baseline| / |μ_sample − μ_baseline|`，仅评估有响应的传感器。按RR分级统计（优秀<5%、良好5-15%、较差≥25%），优+良比例≥80%评为清洗充分。检测过度清洗/漂移传感器并附带具体传感器名报告。

**流程图：**
```
sample_stage = self.results['sample_stage']
    │
    ├─ sample_stage == 'P15':
    │     clean_stage = 'P16' if P16存在 else 'P18' if P18存在 else None
    │
    ├─ sample_stage == 'P17':
    │     clean_stage = 'P18' if P18存在 else None
    │
    ├─ clean_stage 为 None → 跳过
    │
    ├─ extract_cleaning_steady_state(clean_data, baseline) → clean_steady
    │
    ├─ 残留率计算 (仅评估 responsive_sensors):
    │     recovery_rate = |μ_clean - μ_baseline| / |μ_sample - μ_baseline|
    │     若 residual × response_amplitude < 0 → 标记为过度清洗
    │
    ├─ 分级统计:
    │     优秀(RR<5%) + 良好(RR<15%) ≥ 80% → +10分
    │     平均RR ≤ 15% → +5分
    │     否则 → -10分
    │
    └─ 漂移检测: 越过基线且偏离>响应幅度50%的传感器点名警告
```

**入口参数：** 无（从 `self.results` 读取）。

**出口参数：** 写入 `self.results`：

| 字段 | 说明 |
|------|------|
| `cleaning_stage` | 匹配到的清洗阶段名 |
| `cleaning['recovery_rate']` | 各传感器残留率 |
| `cleaning['avg_recovery']` | 平均残留率 |
| `cleaning_degree` | 综合清洗度 `(1-avg_recovery)×100`（可为负） |

**调用方式：**
```python
eda.analyze_cleaning()
```

### 4.8 `_calculate_discrimination(self)`

**功能说明：**
高多样性场景下的区分度计算核心。对每个传感器：
1. 按气体+浓度分组，计算组间最大差异（信号项 `Max_diff`）和组内平均标准差（噪声项 `Avg_std`）
2. 计算 `SNR = Max_diff / (Avg_std + ε)`
3. 单因素ANOVA计算F值和p值
4. **95分位数自适应归一化**：`SNR_norm = min(SNR / P95(SNR), 1.0)`，`F_norm = min(F / P95(F), 1.0)`
5. 综合评分 `Score = 0.6 × SNR_norm + 0.4 × F_norm`
6. 按Score降序排列，给出保留/剔除建议

**入口参数：** 无（从 `self.P15_matrix` 和 `self.file_metadata` 读取）。

**出口参数：**

| 返回 | 类型 | 说明 |
|------|------|------|
| `df` | `pd.DataFrame` | 含 sensor_idx, sensor_name, snr, f_value, p_value, snr_norm, f_norm, score, recommendation |

### 4.10 `calculate_correlation(self)`

**功能说明：**
传感器相关性分析主入口。根据多样性模式分流：
- **低多样性**：随机抽取10个样本，计算每个样本P15动态阶段（前50%数据）的时序相关系数矩阵，取10个样本的平均
- **高多样性**：基于所有样本的稳态响应值（跨样本）计算相关系数矩阵

两种模式均以 |r| > 0.8 为高相关阈值。

**入口参数：** 无。

**出口参数：** 写入 `self.results['correlation']`。

**调用方式：**
```python
eda.calculate_correlation()
```

### 6.5 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `[ERROR] 数据文件列数不足` | 当前列数小于51 | 检查是否误传了非数据文件、文件是否被截断 |
| `[WARNING] 未找到进样阶段(P15或P17)数据` | 数据中无P15/P17标签 | 确认实验流程是否产生了进样阶段，或检查第5列标签值 |
| 中文图表显示方框 | 缺少SimHei字体 | 安装黑体字体或修改 `config.py` 中的 `FONT_PATH` |
| 评分偏低（<60） | 基线波动、响应弱等多因素叠加 | 查看报告中的 `[WARNING]` 列表逐项排查 |
| `找不到文件` 或路径报错 | 路径含括号/空格等特殊字符 | 用英文双引号包裹完整路径 |
